# Documentation Standards

이 디렉터리는 구현 전에 합의해야 하는 공통 문서 템플릿과 작성 기준을 관리한다.

## Document Flow

문서 작성 흐름은 다음 순서를 기본으로 한다.

1. `SPEC.md`: 문제, 범위, 제약, 성공 기준을 확정한다.
2. `design.md`: 구현 구조와 기술 선택을 구체화한다.
3. `task.md`: 실제 작업 단위와 실행 순서를 관리한다.

## Rules

- 새 기능 또는 시스템 작업은 먼저 `docs/SPEC.md`를 기준으로 명세를 작성한다.
- 각 섹션이 해당되지 않더라도 비워 두지 말고 `N/A`와 이유를 기록한다.
- 성공 기준은 반드시 검증 가능하고 측정 가능해야 한다.
- `In Scope`와 `Out of Scope`는 구현 전에 확정한다.
- 설계와 작업 문서는 명세 문서와 모순되지 않아야 한다.
