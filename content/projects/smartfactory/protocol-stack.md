# Protocol Stack

Smart Factory는 단일 프로토콜 환경을 가정하지 않았습니다. 현장 시스템은 보통 하나의 표준만으로 설명되지 않기 때문입니다. 이 문서는 수집 계층을 왜 다중 프로토콜 기준으로 잡았는지 설명합니다.

## 수집 계층

- OPC UA는 산업 장비와의 표준화된 연결 축입니다.
- MQTT는 경량 이벤트 전송과 브로커 기반 연결에 적합한 축입니다.

## 프로토콜을 둘 다 본 이유

- 실제 장비 환경은 항상 균일하지 않습니다.
- 표준 인터페이스 중심 장비와 경량 메시지 기반 흐름이 같이 존재할 수 있습니다.
- 한 가지 프로토콜만 상정하면 전체 시스템 설계가 지나치게 단순화됩니다.

## 왜 둘 다 봤는가

- 장비 환경이 항상 균일하지 않았습니다.
- 실제 현장에서는 표준 인터페이스와 경량 메시지 기반 흐름이 함께 존재할 가능성이 높았습니다.

## 구현 관점에서 남는 의미

- 수집기는 프로토콜 어댑터 계층으로 분리하는 편이 맞습니다.
- 이후 파이프라인은 입력 소스보다 공통 데이터 모델에 집중할 수 있습니다.
- 대시보드와 저장소는 프로토콜 세부사항보다 정규화된 이벤트를 기준으로 설계할 수 있습니다.

## 구현 의미

- 수집기는 프로토콜을 숨기지 않고 어댑터 계층으로 나누는 편이 맞았습니다.
- 이후 저장과 대시보드 계층은 수집 소스보다 공통 데이터 흐름에 집중할 수 있습니다.

## 아직 열려 있는 질문

- 어떤 프로토콜 조합이 실제 현장 비용에 가장 잘 맞는지
- 어댑터 계층을 어디까지 일반화할지
- 수집 복잡도를 운영 복잡도로 그대로 넘기지 않으려면 무엇을 더 추상화해야 할지
