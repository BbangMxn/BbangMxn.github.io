Java Object Research는 Java 문법을 설명하는 문서 묶음이 아니라, JVM의 객체 모델이 런타임 비용과 설계 선택에 어떤 제약을 만드는지 추적하는 연구 묶음입니다.

## 왜 이 연구를 했는가

GC, 힙 기반 객체 모델, 참조 중심 레이아웃은 Java의 기본 전제이지만, 실제 시스템 비용을 설명할 때는 자주 뒷순위로 밀립니다. 이 연구는 JVM 구조를 먼저 이해한 뒤, 그 구조가 객체 오버헤드와 설계 판단에 어떤 영향을 주는지 순서대로 확인하기 위해 시작했습니다.

## 먼저 잡아둘 핵심 관찰

1. JVM 구조를 모르면 객체 비용을 설명할 수 없습니다.
2. 객체 헤더, 참조 추적, 배열 구조는 실제 데이터보다 큰 고정 오버헤드를 만들 수 있습니다.
3. 이 연구는 문법 학습이 아니라 JVM 한계와 대안 설계 가능성을 보는 흐름입니다.
