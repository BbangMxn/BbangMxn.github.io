Java는 객체지향적 추상화를 통해서 높은 생산성과 안전성을 제공한다. 그러나 이러한 객체모델은 과도한 오버헤드를 발생하게 되고 자동적인 메모리 오버헤드가 발생한다.

## 1. JAVA의 객체란?
Java에서 객체는 단순히 new로 만드는 물건이 아니라 state, type,reference, lifecylce를 가진 런타임 엔티티다. 
- 클래스 인스턴스는 객체이다
- 배열 역시 객체이다.
- 객체는 실행 중 Class 메타데이터와 연결된다.
- 대부분의 객체는 Heap에 배치되며, 참조를 통해 접근한다.
Java에서 객체는 단순히 `new`로 생성되는 값이 아니라,  
**state, type, reference, lifecycle**을 가진 런타임 엔티티이다

-----
## 2. 객체 오버헤드 
Java의 객체에서 오버헤드
- **헤더 비용**
	- class Pointer
- **참조 추적 비용**
	- 객체가 직접 값을 담기보다 참조를 통해 연결되는 경우가 많음
	- Pointer chasing
- **배치 문제**
- **GC 대상 증가**
	- 객체 수가 많아질수록 GC pressure 증가
------
### 3. 다양한 JVM에서 개선
### HotSpot
Object Header 크기를 줄이는 **Compact Object Headers**를 jdk 24에서 실험 기능으로 도입
JDK 25에서는 제품 기능으로 올렸다

### Valhalla
OpenJDK는 value objects를 통해 **객체지향적 추상화는 유지하되, primitive에 가까운 성능 특성**을 얻으려는 방향을 공개적으로 연구 중이다. draft JEP에서도 value objects는 `final` 필드만 가지며 **object identity가 없는 인스턴스**로 설명된다.

## 4. 실험 1: 단순 객체의 크기 측정

```java
import org.openjdk.jol.info.ClassLayout;

public class ObjectTest {
    static class EmptyObject {
    }

    static class OneInt {
        int a;
    }

    static class TwoInt {
        int a;
        int b;
    }

    public static void main(String[] args) {
        EmptyObject emptyObject = new EmptyObject();
        OneInt oneInt = new OneInt();
        TwoInt twoInt = new TwoInt();

        System.out.println("EmptyObject");
        System.out.println(ClassLayout.parseInstance(emptyObject).toPrintable());

        System.out.println("OneInt");
        System.out.println(ClassLayout.parseInstance(oneInt).toPrintable());

        System.out.println("TwoInt");
        System.out.println(ClassLayout.parseInstance(twoInt).toPrintable());
    }
}

```

## 5. 실험 2: 단순 객체의 크기 측정

```java
import org.openjdk.jol.info.ClassLayout;
import org.openjdk.jol.info.GraphLayout;

public class ObjectTest {

    static class EmptyObject {}

    static class OneInt {

        int[] a = new int[10];
    }

    static class TwoInt {

        int[] a = new int[10];
        int[] b = new int[10];
    }

    public static void main(String[] args) {
        EmptyObject emptyObject = new EmptyObject();
        OneInt oneInt = new OneInt();
        TwoInt twoInt = new TwoInt();

        System.out.println("=== ClassLayout ===");
        System.out.println("EmptyObject");
        System.out.println(
            ClassLayout.parseInstance(emptyObject).toPrintable()
        );

        System.out.println("OneInt");
        System.out.println(ClassLayout.parseInstance(oneInt).toPrintable());

        System.out.println("TwoInt");
        System.out.println(ClassLayout.parseInstance(twoInt).toPrintable());

        System.out.println("int[10] from OneInt.a");
        System.out.println(ClassLayout.parseInstance(oneInt.a).toPrintable());

        System.out.println("int[10] from TwoInt.a");
        System.out.println(ClassLayout.parseInstance(twoInt.a).toPrintable());

        System.out.println("=== GraphLayout ===");
        System.out.println(
            "EmptyObject total size: " +
                GraphLayout.parseInstance(emptyObject).totalSize() +
                " bytes"
        );

        System.out.println(
            "OneInt total size: " +
                GraphLayout.parseInstance(oneInt).totalSize() +
                " bytes"
        );

        System.out.println(
            "TwoInt total size: " +
                GraphLayout.parseInstance(twoInt).totalSize() +
                " bytes"
        );
    }
}

```

## 6.3 실제 데이터 대비 오버헤드

`int[10]` 배열의 실제 데이터 크기는 다음과 같다.

10 × 4B = 40B

### `int[10]` 자체 오버헤드
- 총 크기: 56B
- 실제 데이터: 40B
- 오버헤드: 16B
- 오버헤드율: **40%**
### `OneInt` 전체 오버헤드

- 실제 데이터: 40B
- 전체 크기: 72B
- 오버헤드: 32B
- 오버헤드율: **80%**
### `TwoInt` 전체 오버헤드
- 실제 데이터: 80B
- 전체 크기: 136B
- 오버헤드: 56B
- 오버헤드율: **70%**
